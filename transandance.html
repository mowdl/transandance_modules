ft_transcendence Project Outline

Version 16.0 Summary

I. Mandatory Part

Minimum requirements for the project website.

A. Overview

  • Create a website for the Pong contest.
  • Must offer a nice user interface.
  • Must include real-time multiplayer capabilities for Pong.

B. Minimal Technical Requirements

  • Backend:
    • Optional.
    • If included, must be pure PHP without frameworks (can be overridden by the Framework module).
    • If a database is used, must follow the Database module constraints (SQLite).
  • Frontend:
    • Must use Typescript as the base code (can be modified by the FrontEnd module - Tailwind CSS + Typescript).
  • Website Architecture:
    • Must be a Single-Page Application (SPA).
    • Browser's Back and Forward buttons must function correctly.
  • Compatibility: Must be compatible with the latest stable version of Mozilla Firefox.
  • Error Handling: No unhandled errors or warnings when browsing the website.
  • Deployment:
    • Must use Docker.
    • Everything must launch with a single command line (e.g., docker run ... or docker-compose up).
    • Be aware of potential rootless Docker constraints on campus computers (runtime location, bind-mounts).

C. Game (Pong)

  • Core Gameplay: Live Pong game against another player on the same keyboard (can be enhanced with the "Remote players" module).
  • Tournament System:
    • Must support multiple players.
    • Must clearly display who is playing whom and the order of play.
    • Requires a matchmaking system to organize participants and announce matches.
  • Registration:
    • Basic registration system required: players input an alias at the start of a tournament.
    • Aliases reset when a new tournament begins.
    • (Can be modified/replaced by the "Standard User Management" module).
  • Fairness: All players (including AI, if implemented) must have identical paddle speeds.
  • Implementation:
    • Must adhere to default frontend constraints (Typescript) OR use the FrontEnd module (Tailwind CSS + Typescript) OR the Graphics module (Babylon.js).
    • Must capture the essence of the original Pong (1972), though visual aesthetics can vary.

D. Security Concerns

  • Password Storage: Any stored passwords must be hashed.
  • Vulnerabilities: Must be protected against SQL injections and XSS attacks.
  • Encryption: HTTPS connection is mandatory for all aspects if a backend or other features requiring secure connections are used (e.g., use wss instead of ws).
  • Input Validation: Must implement validation for forms and any user input (client-side if no backend, server-side if backend exists).
  • General Security: Prioritize website security (e.g., protecting API routes) even if not implementing the JWT/2FA module.
  • Credentials: No credentials, API keys, etc., should be stored in the git repository (use .env files).

II. Modules

A minimum of 7 Major Modules is required for 100% completion.
(Note: 2 Minor Modules count as 1 Major Module).

A. Web

Major Use Fastify with Node.js for the Backend
Use Fastify with Node.js. (Overrides default PHP requirement if backend is used).
Minor Use Tailwind CSS with Typescript for the Frontend
Use Tailwind CSS in addition to Typescript. (Overrides default frontend constraints).
Minor Use SQLite for the Database
Use SQLite for all DB instances. (May be prerequisite for other modules like Backend Framework).
Major Store Scores on Avalanche Blockchain using Solidity
Implement secure storage of tournament scores on a testing blockchain (Avalanche) using Solidity smart contracts. Requires integration with the website for recording, managing, and retrieving scores. May require backend adjustments.

B. User Management

Major Implement Standard User Management (Auth, Profile, Friends, History)
Implement secure user subscription, login, unique display names, profile updates (info, avatar with default), friends list with status, profile stats (wins/losses), and match history (1v1, dates, details). Handle duplicate usernames/emails logically. (Replaces basic alias registration).
Major Implement Remote Authentication with Google Sign-in
Integrate Google Sign-in. Obtain necessary credentials, implement user-friendly and secure login/authorization flows, and ensure secure token/info exchange. Handle potential duplicate usernames/emails logically.

C. Gameplay and User Experience

Major Enable Remote Multiplayer Pong
Allow two players on separate computers to play the same Pong game via the website. Handle network issues (disconnections, lag) gracefully for the best user experience. (Enhances mandatory same-keyboard play).
Major Enable Multiplayer Pong (>2 Players)
Support more than two players (e.g., 3, 4, 5, 6+) in a single game. Requires live control (Remote Players module recommended). Define game rules/board for the chosen number of players (e.g., 4 players on a square board).
Major Add Another Game with History & Matchmaking
Develop a new game distinct from Pong. Include user history tracking (gameplay stats) and a matchmaking system for the new game. Ensure secure storage and optimize performance.
Minor Add Game Customization Options (Power-ups, Maps)
Provide customization options (e.g., power-ups, attacks, maps) applicable to all available games. Allow users to choose a default/basic version. Implement user-friendly settings and maintain consistency across games.
Major Implement Live Chat (DMs, Blocks, Invites)
Create a chat feature allowing:
  • Sending direct messages to other users.
  • Blocking other users (hides their messages).
  • Inviting users to play Pong via chat.
  • Tournament system notifications for next games.
  • Accessing player profiles via chat.

D. AI-Algo

Major Introduce a Challenging AI Opponent (No A*)
Develop a challenging AI opponent for Pong. AI must simulate human keyboard input (refreshing view only once per second, requiring anticipation). AI must use power-ups if the Customization module is implemented. **A\* algorithm is NOT permitted.** Explain AI logic during evaluation; AI must be capable of winning.
Minor Create User and Game Stats Dashboards
Create dashboards displaying individual user stats and game session details (outcomes, historical data). Use data visualization (charts, graphs). Allow users to explore their history/performance. Include any useful metrics.

E. Cybersecurity

Major Implement WAF/ModSecurity & HashiCorp Vault
Configure and deploy a Web Application Firewall (WAF) and ModSecurity with a hardened configuration. Integrate HashiCorp Vault for secure management of secrets (API keys, credentials, env vars).
Minor Implement GDPR Compliance Options (Anonymization, Deletion)
Implement features allowing users to: request anonymization of personal data, manage local data (view/edit/delete), and request permanent account/data deletion. Provide clear communication and options. Familiarize with GDPR principles.
Major Implement 2FA and JWT Security
Implement 2FA (e.g., SMS, authenticator app, email codes) as a secondary verification layer. Utilize JSON Web Tokens (JWT) for secure authentication, authorization, and session management. Ensure secure token handling and user-friendly 2FA setup.

F. Devops

Major Set up ELK Stack for Log Management
Set up the ELK Stack (Elasticsearch, Logstash, Kibana) for log management. Configure Elasticsearch for storage/indexing, Logstash for collection/processing, and Kibana for visualization/dashboards. Define retention policies and secure access.
Minor Set up Prometheus & Grafana Monitoring
Set up Prometheus for metrics collection/alerting and Grafana for visualization. Configure data exporters, create custom dashboards, set up alerting rules, manage data retention, and secure Grafana access.
Major Design Backend as Microservices
Architect the backend using microservices. Divide into loosely-coupled services with clear boundaries/interfaces. Implement inter-service communication (e.g., REST APIs, message queues). Each service should handle a specific function for maintainability and scalability.

G. Graphics

Major Implement Advanced 3D Pong with Babylon.js
Enhance Pong's visuals using advanced 3D techniques with Babylon.js to create an immersive experience. (Overrides default/Frontend module implementation for Pong).

H. Accessibility

Minor Ensure Responsive Support on All Devices
Ensure the website is responsive (adapts to different screen sizes/orientations) and works with various input methods (touch, keyboard, mouse) for a consistent experience across desktops, laptops, tablets, and smartphones.
Minor Expand Browser Compatibility
Add support for one additional web browser beyond the mandatory latest stable Firefox. Conduct testing and optimization to ensure correct function and display in the added browser.
Minor Implement Multiple Language Support (Min. 3)
Implement support for a minimum of three languages. Provide a language switcher. Translate essential content. Ensure seamless navigation regardless of language. Allow users to set a default language.
Minor Add Accessibility for Visually Impaired Users
Implement features like screen reader support, descriptive alt text for images, high-contrast color scheme, keyboard navigation/focus management, and text size adjustment options. Meet accessibility standards.
Minor Integrate Server-Side Rendering (SSR)
Integrate SSR to improve loading speed and SEO. Ensure content is pre-rendered on the server. Maintain a consistent user experience.

I. Server-Side Pong

Major Implement Server-Side Pong & API
Develop server-side logic for Pong (gameplay, movement, scoring, interactions). Create an API with endpoints for game interaction (initialization, controls, state updates) accessible via CLI and web. Integrate with the web application.
Major Enable CLI vs. Web Pong Gameplay via API
Develop a CLI application for playing Pong. Use the API (from the previous module - strongly recommended) to connect the CLI to the web application, allowing CLI users to play against web users. Implement CLI authentication and real-time synchronization. Provide documentation.

III. Bonus Part

  • Bonus points awarded for additional modules beyond the required 7 Major ones.
  • 5 points per extra Minor Module.
  • 10 points per extra Major Module.
  • Crucially: The bonus part is only assessed if the Mandatory Part is PERFECT.
